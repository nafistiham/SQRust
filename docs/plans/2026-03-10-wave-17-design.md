# Wave 17 — Design Doc

Date: 2026-03-10
Status: Approved

## Overview

Add 10 new lint rules across all 5 categories (2 per category). All rules are distinct from the existing 155 rules, sourced from sqlfluff's stable catalog and common SQL anti-pattern databases.

## Rules

### Convention

| Rule | sqlfluff ref | Description |
|------|-------------|-------------|
| `ExplicitAlias` | AL01 | Table aliases in FROM/JOIN must use `AS` keyword |
| `OrInsteadOfIn` | — | Three or more `col = x OR col = y` on the same column → use `IN()` |

### Lint

| Rule | sqlfluff ref | Description |
|------|-------------|-------------|
| `ColumnAliasInWhere` | AL04 | Column alias defined in SELECT referenced in WHERE clause (invalid ANSI SQL) |
| `DuplicateJoin` | — | Same table name joined more than once in the same FROM clause |

### Structure

| Rule | sqlfluff ref | Description |
|------|-------------|-------------|
| `WildcardInUnion` | ST07 | `SELECT *` in any branch of a `UNION` / `INTERSECT` / `EXCEPT` |
| `UnqualifiedColumnInJoin` | ST10 | Column references not qualified with a table/alias in a multi-table `FROM` |

### Ambiguous

| Rule | sqlfluff ref | Description |
|------|-------------|-------------|
| `FloatingPointComparison` | — | Exact `=` or `!=` / `<>` comparison against a floating-point literal |
| `AmbiguousDateFormat` | — | Date string literal in slash-separated format (`'MM/DD/YYYY'`) — locale-dependent |

### Layout

| Rule | sqlfluff ref | Description |
|------|-------------|-------------|
| `ArithmeticOperatorPadding` | LT03 | Arithmetic operators (`+`, `-`, `*`, `/`, `%`) must be padded with spaces |
| `BlankLineAfterCte` | — | Consecutive CTE definitions must be separated by a blank line |

## Implementation Pattern

Same as Waves 1–16:
- One file per rule: `sqrust-rules/src/<category>/<rule_name>.rs`
- TDD: test file first (RED), then implementation (GREEN)
- ≥13 tests per rule
- Register in category `mod.rs` and `sqrust-cli/src/main.rs`
- Atomic commit per category

## Parallelism

5 agents dispatched simultaneously — one per category. Each agent owns its `mod.rs` and touches no other category.

## Success Criteria

- All 10 rules implemented with ≥13 tests each
- `cargo test --workspace` passes with 0 failures
- Total rule count: 165
