# Wave 16 — Design Doc

Date: 2026-03-10
Status: Approved

## Overview

Add 10 new lint rules across all 5 categories (2 per category). All rules sourced from sqlfluff's stable catalog — battle-tested definitions, well-understood edge cases.

## Rules

### Convention

| Rule | sqlfluff ref | Description |
|------|-------------|-------------|
| `LeftJoin` | CV08 | Prefer `LEFT JOIN` over `RIGHT JOIN` |
| `JoinConditionStyle` | CV12 | Join filter conditions must use `ON`, not `WHERE` |

### Lint

| Rule | sqlfluff ref | Description |
|------|-------------|-------------|
| `UnusedTableAlias` | AL05 | Table alias defined but never referenced |
| `ConsecutiveSemicolons` | ST12 | Multiple consecutive `;;` |

### Structure

| Rule | sqlfluff ref | Description |
|------|-------------|-------------|
| `NestedCaseInElse` | ST04 | `CASE` nested inside `ELSE` clause — flatten it |
| `UnusedJoin` | ST11 | Joined table whose columns never appear in `SELECT`, `WHERE`, or `HAVING` |

### Ambiguous

| Rule | sqlfluff ref | Description |
|------|-------------|-------------|
| `AmbiguousUnion` | AM02 | `UNION` without `ALL` or `DISTINCT` |
| `InconsistentColumnReference` | AM06 | Mixing positional and named refs in the same `ORDER BY` / `GROUP BY` |

### Layout

| Rule | sqlfluff ref | Description |
|------|-------------|-------------|
| `SelectTargetNewLine` | LT09 | Each `SELECT` column on its own line when there are multiple |
| `SetOperatorNewLine` | LT11 | `UNION` / `INTERSECT` / `EXCEPT` surrounded by newlines |

## Implementation Pattern

Same as Waves 1–15:
- One file per rule: `sqrust-rules/src/<category>/<rule_name>.rs`
- TDD: test file first (RED), then implementation (GREEN)
- ≥13 tests per rule
- Register in category `mod.rs` and `sqrust-cli/src/main.rs`
- Atomic commit per category

## Parallelism

5 agents dispatched simultaneously — one per category. Each agent owns its `mod.rs` and touches no other category.

## Success Criteria

- All 10 rules implemented with ≥13 tests each
- `cargo test --workspace` passes with 0 failures
- Total rule count: 155
